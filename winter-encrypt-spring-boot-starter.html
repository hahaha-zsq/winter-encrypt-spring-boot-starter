项目简介
Declarative Crypto for Spring Boot

Winter-Encrypt 是一个基于 AOP（面向切面编程）的 Spring Boot 加解密工具包。通过在控制器方法的参数和返回值上添加注解，即可实现对敏感字段的自动加解密，无需编写任何加解密逻辑。

底层集成 Hutool 加密工具与 BouncyCastle 密码库，支持国际标准算法（AES、DES、RSA）和中国国家密码算法（SM2、SM3、SM4），同时支持复杂容器类型（List、Set、Map、Array）中元素级的批量处理。

自 v0.0.1 发布以来，经过 v0.0.30（新增解密能力）、v0.0.31（策略模式重构 + 容器支持）、v0.0.32（RSA 安全增强）等迭代，已形成完整的声明式加解密方案。

能力详情
AES 对称加密
高级加密标准，支持 128/192/256 位密钥。
  • 五种模式：ECB, CBC, CFB, OFB, CTR
  • 三种填充：PKCS5Padding, NoPadding, ISO10126Padding
  • 密钥长度：16 / 24 / 32 字节
  • IV 长度：16 字节（支持 auto-adjust）
DES 对称加密
数据加密标准，64 位密钥，适合向后兼容场景。
  • 五种模式：ECB, CBC, CFB, OFB, CTR
  • 三种填充：PKCS5Padding, NoPadding, ISO10126Padding
  • 密钥长度：8 字节（64 位）
  • IV 长度：8 字节（支持 auto-adjust）
RSA 非对称加密
Rivest-Shamir-Adleman 算法，适用于密钥分发。
  • 算法：RSA/ECB/PKCS1Padding
  • 私钥格式：PKCS#8 Base64（无 PEM 头）
  • 公钥格式：X.509 Base64（无 PEM 头）
  • RSA-1024 最大加密 ~117 字节
SM2 / SM3 / SM4 国密
中国国家密码标准算法实现。
  • SM2：椭圆曲线加解密 + 签名验签
  • SM3：密码杂凑（类似 SHA-256）
  • SM4：分组加密，ECB + CBC 模式
  • 基于 BouncyCastle 实现
注解驱动加解密
声明式编程，零侵入集成。
  • @Encrypt：拦截返回值加密
  • @Decrypt：拦截请求参数解密
  • @FieldEncrypt：标记待加密字段
  • @FieldDecrypt：标记待解密字段
容器批量处理
集合与数组元素级自动处理。
  • List<String> / Set<String> 自动遍历
  • Map<String,String> 仅加密 value
  • String[] 数组逐元素处理
  • >50 元素自动 Stream 并行
静态工具 CryptoUtil
20+ 静态方法，直接调用。
  • AES / DES / RSA / SM2/SM3/SM4
  • 凯撒密码（winterEncryptKaiser）
  • MD5 / SHA1 哈希
  • 密钥生成工具方法
Key/IV 自动适配
智能修正密钥与向量长度。
  • auto-adjust-iv 配置开关
  • 开启后自动截断或补零
  • 关闭则严格校验长度
  • CryptoValidator 集中管理
注解参考
Annotation Reference

四个核心注解构成完整的声明式加解密体系。方法级注解控制加解密时机，字段级注解标记目标字段并配置算法参数。

注解目标属性说明
@EncryptMethod—标注在 Controller 方法上，方法执行后自动加密返回值中所有带 @FieldEncrypt 的字段
@DecryptMethod—标注在 Controller 方法上，方法执行前自动解密请求参数中所有带 @FieldDecrypt 的字段
@FieldEncryptFieldcryptoType, mode, padding标记在 VO/DTO 的 String 字段上，指定加密算法与参数。支持容器类型字段（List/Set/Map/String[]）
@FieldDecryptFieldcryptoType, mode, padding标记在 VO/DTO 的 String 字段上，指定解密算法与参数。支持容器类型字段

@FieldEncrypt / @FieldDecrypt 属性详情

属性类型默认值可选值

modeMode (Hutool)Mode.CBCECB, CBC, CFB, OFB, CTR

使用示例：

注解组合示例
// 响应加密 — AES-CBC 模式 @Encrypt @GetMapping("/user/{id}") public UserVO getUser(@PathVariable Long id) { ... } // 请求解密 — RSA 非对称 @Decrypt @PostMapping("/secure-payload") public Result handleSecure(@RequestBody SecureDTO dto) { ... } // 字段级：AES + ECB 模式 + 无填充 @FieldEncrypt(cryptoType = CryptoType.AES, mode = Mode.ECB, padding = Padding.NoPadding) private String cardNumber; // 字段级：RSA 加密 @FieldEncrypt(cryptoType = CryptoType.RSA) private String ssn; // 字段级：List 容器 AES 加密 @FieldEncrypt(cryptoType = CryptoType.AES) private List<String> phoneNumbers; // 解密字段：DES + CFB 模式 @FieldDecrypt(cryptoType = CryptoType.DES, mode = Mode.CFB) private String secretKey;
配置参考
Configuration Reference

所有配置前缀为 winter-crypto。仅 aes.key 和 aes.iv 为必配项，缺一不可 — 缺少任意一项将导致整个自动配置被跳过。

属性类型默认值说明
aes.keyString—AES 密钥，16/24/32 字节十六进制字符串
aes.ivString—AES 初始化向量，16 字节字符串
aes.auto-adjust-ivBooleantrue自动调整 IV 长度（截断或补零至 16 字节）
des.keyString—DES 密钥，8 字节字符串（可选）
des.ivString—DES 初始化向量，8 字节字符串（可选）
des.auto-adjust-ivBooleantrue自动调整 DES IV 长度至 8 字节
rsa.private-keyString—RSA 私钥，PKCS#8 格式 Base64 编码（可选）
rsa.public-keyString—RSA 公钥，X.509 格式 Base64 编码（可选）
is-printBooleantrue是否在启动时打印 Winter ASCII 横幅

完整配置示例：

application.yml
winter-crypto: aes: key: 346a3f9f4c1988cb7a507cc177923ac7 iv: 1234567887654321 auto-adjust-iv: true des: key: 12345678 iv: 87654321 auto-adjust-iv: true rsa: private-key: "${RSA_PRIVATE_KEY}" public-key: "${RSA_PUBLIC_KEY}" is-print: true

安全检查：生产环境务必使用环境变量注入密钥，禁止硬编码。示例中的 ${RSA_PRIVATE_KEY} 将在运行时从环境变量读取。

使用流程
Getting Started

三步集成，无需额外配置。从添加依赖到注解完成，全程零 Java Config。

1
添加 Maven 依赖
在 pom.xml 中添加 winter-encrypt-spring-boot-starter 依赖。
pom.xml Maven Central
<dependency> <groupId>io.github.hahaha-zsq</groupId> <artifactId>winter-encrypt-spring-boot-starter</artifactId> <version>0.0.32</version> </dependency>
2
配置 application.yml
设置 AES 密钥与 IV（必配）。DES 和 RSA 为可选配置。生产环境使用环境变量注入。
application.yml
winter-crypto: aes: key: 346a3f9f4c1988cb7a507cc177923ac7 iv: 1234567887654321 auto-adjust-iv: true rsa: private-key: "${RSA_PRIVATE_KEY}" public-key: "${RSA_PUBLIC_KEY}"
3
注解控制器与 VO
在 Controller 方法上添加 @Encrypt / @Decrypt，在字段上添加 @FieldEncrypt / @FieldDecrypt。
MyController.java
@RestController @RequestMapping("/api") public class MyController { @Encrypt @GetMapping("/user/{id}") public UserVO getUser(@PathVariable Long id) { UserVO vo = userService.findById(id); return vo; // password, ssn 自动加密 } @Decrypt @PostMapping("/order") public Result createOrder(@RequestBody OrderDTO dto) { orderService.create(dto); return Result.ok(); } } @Data public class UserVO { private String username; @FieldEncrypt(cryptoType = CryptoType.AES, mode = Mode.CBC) private String password; @FieldEncrypt(cryptoType = CryptoType.RSA) private String ssn; @FieldEncrypt(cryptoType = CryptoType.AES) private List<String> phoneNumbers; } @Data public class OrderDTO { private String orderId; @FieldDecrypt(cryptoType = CryptoType.AES, mode = Mode.CBC) private String paymentToken; }

验证集成：启动应用，控制台将打印 Winter ASCII 横幅（is-print: true 时）。使用 POST 请求发送已加密的 JSON 到 @Decrypt 接口，验证数据是否被正确解密。

注意事项
Important Notes

使用前务必了解以下约束条件，避免在生产环境中出现问题。

  • 不支持嵌套对象 仅支持 String 类型字段和 String 元素容器。嵌套对象（如 List<UserVO>）不会触发加解密，抛出 UNSUPPORTED_DATA_TYPE。这是当前版本的主要功能限制。
  • RSA 密钥格式要求 必须使用纯 Base64 编码的 PKCS#8 私钥 / X.509 公钥。不包含 -----BEGIN PRIVATE KEY----- 等 PEM 头信息。推荐使用 CryptoUtil.winterGenerateRsAKey() 生成兼容格式的密钥。
  • RSA 加密长度限制 RSA-1024 最大加密长度约 117 字节，RSA-2048 约 245 字节。加密大量数据时应使用混合加密方案：AES 加密数据 + RSA 加密 AES 密钥。
  • 空值会抛出异常 标注了 @FieldEncrypt / @FieldDecrypt 的字段如果为 null，将抛出 CryptoException(EMPTY_DATA)。请确保所有敏感字段在前置处理中已被正确赋值。空字符串 "" 可以正常处理。
  • 容器类型限制 仅支持 String 元素类型的容器（List<String> / Set<String> / Map<String,String> / String[]）。Map 仅加密 value，key 保持明文。数组加解密后元素类型为 String[]。
  • 密钥长度严格校验 AES 密钥：16/24/32 字节（128/192/256 位）。AES IV：16 字节。DES 密钥：8 字节。DES IV：8 字节。开启 auto-adjust-iv: true（默认开启）可自动修正 IV 长度，关闭则严格校验。
  • 条件化自动配置 必须同时配置 winter-crypto.aes.key 和 winter-crypto.aes.iv，否则整个自动配置被跳过，所有加解密注解不生效。启动时注意控制台是否有 Winter 横幅输出。
  • 生产环境密钥管理 严禁将密钥硬编码在 YAML 文件中提交到版本控制。推荐使用环境变量注入（如 ${AES_KEY}）、Vault 或 AWS KMS 等密钥管理服务。
  • 线程安全 CryptoAspect 中的服务 Bean 在初始化后不可变，对多线程请求是安全的。但请注意 BouncyCastleProvider 在 CryptoUtil 中每次调用都重新注册，建议后续优化为静态初始化。
  • 版本号不一致 源代码中 CryptoConstants.VERSION_NO 硬编码为 v0.0.1，与 POM 版本 0.0.32 不一致。引用了此常量的场景（如 Banner 输出）可能显示过期版本号。
  • 无自定义加密扩展 加密实现固定为内置算法。当前版本不支持用户注入自定义加密算法或替换默认实现。
